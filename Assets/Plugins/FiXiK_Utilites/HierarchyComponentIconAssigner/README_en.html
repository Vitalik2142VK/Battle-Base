Hierarchy Component Icon Assigner

Version 2.2.0 | Author: FiXiK

This plugin allows you to assign icons to components and display them directly in the Unity Hierarchy window.
Icons now automatically inherit: if you assign an icon to an abstract class or interface, it will appear on all objects that have a derived component.

Features

  • Assign icons to any component (MonoBehaviour, ScriptableObject, standard Unity components).
  • Icon inheritance – base type icon is displayed for all its descendants (including abstract classes).
  • Convenient editor – settings window with drag-and-drop for icons and a dropdown list of types.
  • Enable/disable icon rendering without removing settings.
  • Auto-generated config on first launch.
  • Welcome dialog on first installation.

What's New in Version 2.2.0

  • Support for abstract components – icons can now be assigned to abstract classes (e.g., Behaviour, Collider, MonoBehaviour).
  • Icon inheritance – if an object has no direct match, the plugin checks all parent types and shows the icon of the first matching ancestor.
  • Optimized lookup – for each object in the hierarchy, all components are checked for compatibility with types that have icons.

Previous versions (≤ 2.1.2) only worked with concrete (non‑abstract) types and did not support inheritance.

Installation

Via .unitypackage (recommended)

  1. Download the latest release.
  2. Import the .unitypackage via Assets → Import Package → Custom Package.
  3. Done! The plugin will automatically create a config and show the welcome window.

Manual installation

  1. Copy the HierarchyComponentIconAssigner folder into Assets/Plugins/FiXiK_Utilites/.
  2. Reload Unity (or press Ctrl+R).
  3. Open the window via Tools → Hierarchy Scene Icons (or Иконки в иерархии сцены in the Russian interface).

Quick Start

  1. Open the settings window
    Tools → Hierarchy Scene Icons (or Иконки в иерархии сцены)
  2. Enable rendering (tick Enabled at the top of the window).
  3. Add a new icon
    Press + in the Component Icon List field.
  4. Select a component
    Click the button No component selected → choose any type from the dropdown.
  5. Pick an icon
    Click the grey square → select a texture (or drag one from the Project window).
  6. Result
    In the Hierarchy window, all objects that have the selected component (or its descendants) will display the icon.

Configuration

All settings are stored in the Config.asset file, which is automatically created in one of the following folders:

  • Assets/Plugins/FiXiK_Utilites/HierarchyComponentIconAssigner/Config.asset (priority)
  • Or in the folder where the ConfigLoader.cs script is located.

To reset the settings – delete this file, it will be recreated on the next launch.

Manual editing

You can open Config.asset in the Unity Inspector and adjust:

  • Enabled – turn icon rendering on/off.
  • Component Icon List – the list of [Component Type] → [Icon] pairs.

API (for advanced users)

If you want to control the plugin from your own code (e.g., enable/disable rendering based on some condition), use:

using FiXiK.HierarchyComponentIconAssigner;

// Force refresh the cache and repaint the hierarchy
HierarchyIconDrawer.Reload();

// Get the current config
Config cfg = ConfigLoader.Config;
cfg.Enabled = false; // don't forget to save changes
EditorUtility.SetDirty(cfg);

Note: The API is only available in the Editor (#if UNITY_EDITOR).

Requirements

  • Unity 2020.3 or newer (tested on 2021.3 LTS, 2022.3, 2023.2).
  • Platform: any that supports the Unity Editor (Windows, macOS, Linux).
  • Dependencies: none.

Frequently Asked Questions

Q: Why is the icon not showing for an abstract class?
A: Make sure you have selected a concrete component that is actually attached to the object. An abstract class itself cannot be added to a GameObject, but its descendants will display the icon. Check that the object has a component derived from the selected abstract type.
Q: How can I temporarily disable icons?
A: Uncheck the Enabled box in the settings window.
Q: Can I use icons for interfaces?
A: Yes, interfaces are technically supported (the IsAssignableFrom method works). However, interfaces are not components, so ensure your class inherits MonoBehaviour and implements the interface.
Q: Does the plugin slow down the editor?
A: With a very large number of objects (>2000) and many icon types (>20), you may notice minor delays. In typical projects (up to 500 objects), the difference is negligible. If you experience slowdowns, reduce the number of icon types or disable rendering via Enabled.
