Hierarchy Component Icon Assigner

Версия 2.2.0 | Автор: FiXiK

Этот плагин позволяет назначать иконки компонентам и отображать их прямо в окне Hierarchy (Иерархия) Unity.
Теперь иконки автоматически наследуются: если назначить иконку на абстрактный класс или интерфейс, она будет показываться у всех объектов, имеющих компонент-наследник.

Возможности

  • Назначение иконок любым компонентам (MonoBehaviour, ScriptableObject, стандартные компоненты Unity).
  • Наследование иконок – иконка базового типа отображается у всех его потомков (включая абстрактные классы).
  • Удобный редактор – окно настройки с перетаскиванием иконок и выпадающим списком типов.
  • Включение/отключение отрисовки иконок без удаления настроек.
  • Автоматическое создание конфига при первом запуске.
  • Приветственный диалог при первой установке.

Что нового в версии 2.2.0

  • Поддержка абстрактных компонентов – иконки теперь можно назначать на абстрактные классы (например, Behaviour, Collider, MonoBehaviour).
  • Наследование иконок – если на объекте нет прямого совпадения, плагин проверяет все родительские типы и показывает иконку первого подходящего предка.
  • Оптимизированный поиск – для каждого объекта в иерархии проверяются все компоненты на совместимость с типами, имеющими иконки.

Предыдущие версии (≤ 2.1.2) работали только с конкретными (неабстрактными) типами и не поддерживали наследование.

Установка

Через .unitypackage (рекомендуется)

  1. Скачайте последний релиз.
  2. Импортируйте .unitypackage через Assets → Import Package → Custom Package.
  3. Готово! Плагин автоматически создаст конфиг и покажет приветственное окно.

Ручная установка

  1. Скопируйте папку HierarchyComponentIconAssigner в Assets/Plugins/FiXiK_Utilites/.
  2. Перезагрузите Unity (или нажмите Ctrl+R).
  3. Откройте окно через Tools → Иконки в иерархии сцены

Быстрый старт

  1. Откройте окно настройки
    Tools → Иконки в иерархии сцены
  2. Включите отрисовку (галочка Enabled вверху окна).
  3. Добавьте новую иконку
    Нажмите + в поле Component Icon List.
  4. Выберите компонент
    Нажмите на кнопку Компонент не выбран → выберите любой тип из выпадающего списка.
  5. Выберите иконку
    Кликните по серому квадрату → укажите текстуру (можно перетащить из окна Project).
  6. Результат
    В окне Hierarchy у всех объектов с выбранным компонентом (или его наследниками) появится иконка.

Конфигурация

Все настройки хранятся в файле Config.asset, который автоматически создаётся в одной из папок:

  • Assets/Plugins/FiXiK_Utilites/HierarchyComponentIconAssigner/Config.asset (приоритет)
  • Или в папке, где лежит скрипт ConfigLoader.cs.

Чтобы сбросить настройки – удалите этот файл, при следующем запуске он создастся заново.

Ручное редактирование

Вы можете открыть Config.asset в инспекторе Unity и настроить:

  • Enabled – вкл/выкл отрисовку.
  • Component Icon List – список пар [Тип компонента] → [Иконка].

API (для продвинутых пользователей)

Если вы хотите управлять плагином из своего кода (например, включать/выключать отрисовку по условию), используйте:

using FiXiK.HierarchyComponentIconAssigner;

// Принудительно обновить кэш и перерисовать иерархию
HierarchyIconDrawer.Reload();

// Получить текущий конфиг
Config cfg = ConfigLoader.Config;
cfg.Enabled = false; // не забудьте сохранить изменения
EditorUtility.SetDirty(cfg);

Примечание: API доступно только в редакторе (#if UNITY_EDITOR).

Требования

  • Unity 2020.3 или новее (проверено на 2021.3 LTS, 2022.3, 2023.2).
  • Платформа: любая, поддерживающая редактор Unity (Windows, macOS, Linux).
  • Зависимости: отсутствуют.

Часто задаваемые вопросы

Вопрос: Почему иконка не отображается для абстрактного класса?
Ответ: Убедитесь, что вы выбрали конкретный компонент, который действительно присутствует на объекте. Абстрактный класс сам по себе не может быть добавлен на GameObject, но его наследники будут отображать иконку. Проверьте, что у объекта есть компонент, производный от выбранного абстрактного типа.
Вопрос: Как временно отключить иконки?
Ответ: Снимите галочку Enabled в окне настроек.
Вопрос: Можно ли использовать иконки для интерфейсов?
Ответ: Да, интерфейсы технически поддерживаются (метод IsAssignableFrom работает). Однако интерфейсы не являются компонентами, поэтому убедитесь, что ваш класс наследует MonoBehaviour и реализует интерфейс.
Вопрос: Замедляет ли плагин работу редактора?
Ответ: При очень большом количестве объектов (>2000) и множестве типов иконок (>20) возможны незначительные задержки. В типичных проектах (до 500 объектов) разница незаметна. Если вы заметили замедления, уменьшите количество типов иконок или отключите отрисовку через Enabled.
